# 学习链接收集器

自动收集并整理学习相关的网址到飞书多维表格。

## 工作流程

- 当用户消息包含URL时,自动提取URL(用户文本可能有多个URL，可以先简单测试是否能正常打开再进行下一步)：

1. **确认表格** - 用 list_my_bitables 搜索"网络知识收集目录"
2. **验证结构** - 用 get_bitable_structure 确认字段
3. **获取内容** - 获取网页标题和摘要（见下方工具选择）
4. **推断分类** - 根据内容分析类别
5. **插入记录** - 用 dd_bitable_record 保存

## 工具选择（获取网页内容）

按优先级尝试：

1. stealthy_fetch - 适合高保护网站（微信公众号等），但需 Playwright 已安装
2. get - 通用 HTTP 请求，适合大多数网站
3. web_fetch - 备选方案

如果 stealthy_fetch 报错 Executable doesn't exist，说明 Playwright 未安装，改用 get。

## 飞书多维表格

- **Name**: 网络知识收集目录

## 字段结构与格式

**重要：添加记录时字段值的格式**

| 字段名 | 类型 | 格式说明 |
|--------|------|----------|
| 标题 | Text | 直接传字符串："标题内容" |
| 一级分类 | Text | 直接传字符串："技术开发" |
| 二级分类 | Text | 直接传字符串 |
| 三级分类 | Text | 直接传字符串 |
| 链接 | URL | {"link": "url", "text": "显示文本", "type": "url"} |
| 内容摘要 | Text | 直接传字符串 |

**正确示例（fields_json）：**

``json
{
  "标题": "文章标题",
  "一级分类": "工具效率",
  "二级分类": "AI编程助手",
  "三级分类": "OpenClaw",
  "链接": {
    "link": "https://mp.weixin.qq.com/s/xxx",
    "text": "显示文本",
    "type": "url"
  },
  "内容摘要": "文章摘要内容..."
}
``

**错误格式（不要使用）：**

`json
// 错误：不要用数组包裹文本字段
{
  "标题": [{"text": "文章标题", "type": "text"}],
  "一级分类": [{"text": "技术", "type": "text"}]
}
`

## 分类类别

**一级分类：**
- 技术开发
- 产品设计
- 商业管理
- 个人成长
- 工具效率
- 金融投资
- 其他

**二级/三级分类：** 根据具体内容推断

## 常见错误处理

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| URLFieldConvFail | 链接字段格式错误 | 使用 {"link": "...", "text": "...", "type": "url"} 格式 |
| TextFieldConvFail | 文本字段用了数组格式 | 直接用字符串，不要用 [{\"text\":...}] |
| Executable doesn't exist | Playwright 未安装 | 改用 get 工具获取网页 |
